# Architecture Rules

## Struktur Folder

- `docker/` berisi konfigurasi Docker seperti nginx dan Vue untuk deployment.
- `src/api` berisi fungsi pemanggilan API ke backend dengan pola `*-service.ts` dan `http.ts`.
- `src/assets` berisi asset statis yang dibundle Vite.
- `src/components` berisi komponen Vue reusable dan bagian halaman.
- `src/composable` berisi logic page/component. Script setup page sebaiknya memanggil composable, bukan menampung flow besar.
- `src/constants` berisi konstanta aplikasi seperti header table dan navigasi layout.
- `src/helper` berisi helper umum.
- `src/layouts` berisi layout global yang membungkus `<router-view>`.
- `src/model` berisi interface dan type domain/payload API.
- `src/pages` berisi file halaman yang terhubung ke route.
- `src/plugins` berisi inisialisasi plugin seperti Vuetify, Pinia, auto-import, dan i18n.
- `src/router` berisi konfigurasi router utama dan route guard.
- `src/schemas` berisi schema validasi dan konfigurasi form.
- `src/stores` berisi Pinia store untuk state global.
- `src/styles` berisi style global dan settings theme.

## File Ownership

- Lebih baik punya banyak file yang penting dan spesifik, dengan nama yang jelas merepresentasikan fitur/tanggung jawabnya.
- Hindari satu file besar yang berisi banyak komponen, flow, atau fitur yang tidak related.
- Jika ada script yang sering duplicate, utamakan buat helper.
- Jika refactor atau fitur baru membuat sebuah file mulai membesar dan tidak fokus, pindahkan bagian dengan tanggung jawab jelas ke file baru yang spesifik.

## Type dan Schema

- Jangan buat interface/type di dalam component.
- Taruh interface/type di `src/model` atau file model relevan.
- Hindari tipe `any` sebisa mungkin.
- `src/schemas/` adalah tempat schema validasi dan konfigurasi form.
- Boleh mengubah `src/schemas/` jika task memang menyentuh validasi, initial form values, atau konfigurasi form.
- Jangan mengubah `src/schemas/` untuk perubahan yang tidak terkait dengan validasi/form.
- Tentukan schema validasi yang jelas saat membuat atau mengubah form validation.

## API dan State

- Jika ada call API, gunakan Promise chain (`.then/.catch/.finally`), jangan `async/await`.
- Bagian `catch` cukup `console.error` karena error handling utama sudah di config API.
- Jangan jadikan Pinia sebagai jalur wajib untuk semua call service/API.
- Gunakan Pinia hanya jika ada kebutuhan global state, shared state lintas page/component, cache bersama, atau koordinasi state aplikasi.
- Jika logic dan data hanya dipakai lokal pada satu page/component/composable, panggil service langsung dari composable/page tanpa membungkusnya ke Pinia.
- Hindari store yang hanya menjadi pass-through ke service tanpa kebutuhan state global yang jelas.

## Imports

- Import `vue` dan `vue-router` tidak perlu karena sudah auto import.
